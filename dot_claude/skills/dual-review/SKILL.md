# Dual Review (Claude + Codex 合意レビュー)

2つのAIレビュアーを並列実行し、**両者が合意した指摘のみ**を修正対象として提案する。

## 手順

### Step 1: レビュー対象の特定

引数が指定されている場合はそのファイル/観点を対象とする。
引数がない場合は `git diff --name-only` で変更ファイルを特定する。

### Step 2: 並列レビュー実行

**以下の2つをAgent toolで同時に起動すること（1つのメッセージで2つのAgent呼び出し）**:

#### Agent A: Claude Review

- subagent_type: `general-purpose`
- プロンプト:

```
以下の変更ファイルをレビューしてください。
対象: {変更ファイル一覧}

レビュー観点:
1. バグ・ロジックエラー
2. セキュリティリスク
3. パフォーマンス問題
4. 設計・可読性の問題
5. エッジケースの見落とし

各指摘について以下の形式で出力:
- ファイル: path/to/file
- 行: 該当行番号（わかる場合）
- 種別: bug / security / performance / design / edge-case
- 深刻度: high / medium / low
- 指摘内容: 具体的な説明
- 修正案: 具体的なコード修正

確認や質問は不要です。具体的な指摘のみ出力してください。
```

#### Agent B: Codex Review

- subagent_type: `general-purpose`
- プロンプト: Bash toolでcodexコマンドを実行し、その出力をそのまま返すこと

##### 実行コマンド

```bash
codex exec --full-auto --sandbox read-only --cd <project_directory> "<request>"
```

##### プロンプトのルール

**重要**: codexに渡すリクエストには、以下の指示を必ず末尾に含めること：

> 「確認や質問は不要です。具体的な提案・修正案・コード例まで自主的に出力してください。」

これがないとCodexが確認待ちで停止する場合がある。

##### パラメータ

| パラメータ | 説明 |
|-----------|------|
| `--full-auto` | 完全自動モードで実行（対話なし） |
| `--sandbox read-only` | 読み取り専用サンドボックス（安全な分析用、ファイル変更なし） |
| `--cd <dir>` | 対象プロジェクトのディレクトリ（`git rev-parse --show-toplevel` で取得） |
| `"<request>"` | レビュー依頼内容（日本語可） |

##### リクエスト文の組み立て

以下のテンプレートで組み立てる。`{変更ファイル一覧}` はStep 1で特定したファイルパスを改行区切りで埋める。

```
以下の変更ファイルをレビューしてください。
対象:
{変更ファイル一覧}

レビュー観点:
(1) バグ・ロジックエラー
(2) セキュリティリスク
(3) パフォーマンス問題
(4) 設計・可読性の問題
(5) エッジケースの見落とし

各指摘について以下の形式で出力:
- ファイル: path/to/file
- 行: 該当行番号（わかる場合）
- 種別: bug / security / performance / design / edge-case
- 深刻度: high / medium / low
- 指摘内容: 具体的な説明
- 修正案: 具体的なコード修正

確認や質問は不要です。具体的な提案・修正案・コード例まで自主的に出力してください。
```

##### 使用例

**注意**: 各例では末尾に「確認不要、具体的な提案まで出力」の指示を含めている。

変更ファイルのレビュー:

```bash
codex exec --full-auto --sandbox read-only --cd /Users/202204012/repo/score-library "以下の変更ファイルをレビューしてください。対象: app/routes/stats_.$id_.tsx, server/application/usecases/mahjong/stats/read.stats.usecase.impl.test.ts。レビュー観点: (1) バグ・ロジックエラー, (2) セキュリティリスク, (3) パフォーマンス問題, (4) 設計・可読性の問題, (5) エッジケースの見落とし。各指摘はファイル名・行番号・種別・深刻度・指摘内容・修正案の形式で出力してください。確認や質問は不要です。具体的な提案・修正案・コード例まで自主的に出力してください。"
```

特定観点に絞ったレビュー:

```bash
codex exec --full-auto --sandbox read-only --cd /Users/202204012/repo/score-library "app/screen/stats_.$id.tsx のTypeScript型安全性とReactパフォーマンス（不要な再レンダリング、メモ化の欠如）を重点的にレビューしてください。確認や質問は不要です。具体的な修正案・コード例まで自主的に出力してください。"
```

##### 実行手順

1. Step 1で特定した変更ファイル一覧を取得する
2. プロジェクトディレクトリを `git rev-parse --show-toplevel` で特定する
3. 上記テンプレートに変更ファイル一覧を埋めてリクエスト文を作成する
4. **リクエスト末尾に「確認や質問は不要です。具体的な提案・修正案・コード例まで自主的に出力してください。」が含まれていることを確認する**
5. `codex exec --full-auto --sandbox read-only --cd <dir> "<request>"` を実行する
6. codexの出力をそのまま返す

### Step 3: 初回合意分析

両方の結果を比較し、以下のカテゴリに分類:

1. **合意（両者一致）**: 両者が同じファイル・同じ問題を指摘
2. **片方のみ（Claude）**: Claudeのみが指摘
3. **片方のみ（Codex）**: Codexのみが指摘

合意した指摘はそのまま修正対象とする。片方のみの指摘がある場合はStep 4へ進む。
片方のみの指摘がない場合はStep 4をスキップしてStep 5へ進む。

### Step 4: クロスレビュー

片方のみの指摘について、もう片方のレビュアーに妥当性を評価させる。
**以下の2つをAgent toolで同時に起動すること（1つのメッセージで2つのAgent呼び出し）**:

#### Agent C: Codex → Claudeのみの指摘を評価

- subagent_type: `general-purpose`
- 「Claudeのみの指摘」が存在しない場合はこのAgentを起動しない
- プロンプト: Bash toolでcodexコマンドを実行し、その出力をそのまま返すこと

##### 実行コマンド

```bash
codex exec --full-auto --sandbox read-only --cd <project_directory> "<request>"
```

##### リクエスト文の組み立て

```
以下はコードレビューで指摘された項目です。各指摘が妥当かどうかを、対象コードを実際に読んで評価してください。

{Claudeのみの指摘一覧（ファイル・行・種別・指摘内容・修正案を含む）}

各指摘について以下の形式で出力:
- 指摘: （元の指摘内容を簡潔に）
- 判定: agree / disagree
- 理由: なぜ同意または不同意か

確認や質問は不要です。具体的な判定と理由を自主的に出力してください。
```

#### Agent D: Claude → Codexのみの指摘を評価

- subagent_type: `general-purpose`
- 「Codexのみの指摘」が存在しない場合はこのAgentを起動しない
- プロンプト:

```
以下はコードレビューで指摘された項目です。各指摘が妥当かどうかを、対象コードを実際に読んで評価してください。

{Codexのみの指摘一覧（ファイル・行・種別・指摘内容・修正案を含む）}

各指摘について以下の形式で出力:
- 指摘: （元の指摘内容を簡潔に）
- 判定: agree / disagree
- 理由: なぜ同意または不同意か
```

### Step 5: 最終合意分析

Step 3の初回合意 + Step 4のクロスレビューで合意が得られた指摘を統合し、最終的なカテゴリに分類:

1. **合意（初回一致）**: Step 3で両者が一致した指摘
2. **合意（クロスレビューで追認）**: Step 4で相手方もagreeと判定した指摘
3. **不合意**: クロスレビューでもdisagreeとなった指摘

### Step 6: 結果提示

以下の形式でユーザーに提示:

```
## 合意した指摘（修正推奨）
### 初回合意
- [ ] 指摘1: ...

### クロスレビューで追認
- [ ] 指摘2: （元の指摘者: Claude / Codex）...

## 不合意の指摘（参考）
- 指摘A: （指摘者: Claude, Codex判定: disagree, 理由: ...）
- 指摘X: （指摘者: Codex, Claude判定: disagree, 理由: ...）
```

### Step 7: 修正実行

AskUserQuestionで修正対象の選択を促し、承認された項目のみ修正を実施する。
